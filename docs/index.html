1 Was ist das, HTML?

Die formale Sprache HTML steht für die englische Wörter Hypertext Markup Language. Zieh Dir die Erklärung jedes Wortes rein, welches diese schöne und weltbewegende Abkürzung HTML schmückt.

1.1 Erklärung

Hypertext bedeutet ein Text, dass durch Querverweise mit anderen Texten verbunden wird. Markup kommt vom englischen to mark up, was wortwörtlich markieren heisst. Das Markieren, welches hier allerdings gemeint ist, ist das Markieren von Manuskripten im Verlagsgeschäft. Das Markieren von Manuskripten wurde vom Schriftsetzer und anderen funktionalen Verlagsmitarbeiter benutzt. Das war eine sehr kondensierte und abgekürzte Kommentare und Symbolen, die dem einfachen Manuskript die notwendigen Stilemente anhing. Language ist die Sprache; hier wiederum handelt es sich um eine formale Programmiersprache.

Eine Markup Language ist also eine Programmiersprache, die die Teile eines Dokuments und die gegenseitigen Beziehungen der Teile zueinander beschreibt. Diese Beschreibung nennen wir deskriptiv semantisch, weil die einzelnen Teile nach ihrer Bedeutung benannt werden. Das bedeuten, der Code steht unmittelbar für das was an der Stelle gezeigt werden soll. Deskriptiv heisst, dass HTML ein reine beschreibende Sprache ist.

1.2 Unterstützung durch andere Sprachen

Da HTML lediglich die Struktur einer Web Ressource wie eine Webseite beschreibt, wird sie auf der Darstellungsseite durch die ebenfalls deklarative Programmiersprache Cascading Style Sheets, CSS unterstützt. Auf dem Client läuft darüber hinaus für die Bereitstellung von Logik die vollwertige Programmiersprache Javascript ab.

2 Syntax

Die Syntax einer Programmiersprache ist ähnlich zu sehen, wie der korrekte Satzbau in der Grammatik einer Sprache. Dadurch, dass Programmiersprachen jedoch kontextfrei sind, ist deren Syntax sehr einfach. Das heisst nicht, dass es nicht genügend Möglichkeiten gibt, wie man sich selber in den Fuss schiesst. Selbst für eine kontextfreie Sprache, brauchst Du etwas Disziplin, um sie korrekt schreiben zu können.

Die Programmiersprache HTML beschreibt durch HTML Elementen die verschiedenen semantischen Elemente einer Web Ressource. Um ein HTML Element anzugeben, benutzt man einen Tag. Ein tag ist wie ein Etikett, dass ein bestimmtes Element anzeigt. In den folgenden Abschnitte lernst Du einige Syntaxaspekte von HTML kennen.

Viele der nachfolgenden Codebeispiele lassen sich kopieren und in sogenannte Livecoding Umgebungen im WWW verwenden. Eine dieser Livecoding Umgebungen ist zum Beispiel: https://developer.mozilla.org/en-US/play.

3.3 Attribute

Attribute sind gesetzte Eigenschaften eines HTML Elements. Um den Unterschied zu verstehen, verdeutliche Dir das durch die Haarfarbe. Die Haarfarbe ist eine Eigenschaft aber Haarfarbe Schwarz ist eine gesetzte Eigenschaft, die als Attribut bezeichnet wird.

Die allerwichtigsten Attribute sind einerseits id und andererseits class. Das id Attribut benennt ein HTML Element eindeutig und einmalig. Diese Attribut darf nur einmal vorkommen; zum Beispiel in einer Webseite. Das Attribut class ordnet das HTML Element einer bestimmten Klasse von Objekten zu. Wichtig

Achtung. Code wird in der realen Welt viel öfters gelesen als geschrieben. Dadurch haben Kommentare eine unglaublich große Bedeutung in der praktischen Informatik. Sie helfen den Code zu strukturieren, Änderungen anzukündigen, Funktion zu erklären, und vieles andere. Ein aus Faulheit unkommentierter Code ist in der wahren Welt eine Katastrophe.
